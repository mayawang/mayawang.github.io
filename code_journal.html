    • Priority in coding

    1. correctness/quality 2. clean/readable 3. least amount of code 4. fast development speed(less error handling/structure design/maybe increase maintenance and human resources)

    What is "Quality" code

    June 1st, 2016

    1. adaptable/flexible (will items be changed later? if so, have to design the code to easily accommodate the changes) - array/hash, collection => data - class/method => implementation - API/modularity => functionality 2. avoid complexity of thinking (can I figure out what the method does by just looking at the method name/parameters? If not, the method is not clearly organized/defined.) 3. size of code should be proportional to its functionality. Too much or too little is problematic. 4. Repetitivity can always be replaced by loop (if only the value or variable is changed. method changed is not worth looping yet. behavior/object orientated thinking)
